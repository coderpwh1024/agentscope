一张专业、干净的**流程图（flowchart）**，扁平化矢量信息图风格，浅色背景（淡灰白 #F7F8FA），圆角矩形与菱形判定节点，细描边，柔和阴影，配色为科技蓝 + 绿色 + 中性灰，箭头清晰带文字标签，竖向自上而下布局，含一条明显的循环回环箭头，中文字体清晰可读、排版整齐、无乱码。

主题：ReAct Agent 工作原理（Reasoning + Acting 循环）。

节点（从上到下，竖向流程）：

1. 起始圆角矩形：「收到 Msg」。
2. 箭头向下到方块（标 Reason 阶段）：「(Reason) 调 LLM：依据 sysPrompt + memory + 可用工具列表，决定下一步」。
3. 箭头向下到菱形判定节点：「需要调工具？」。
4. 从菱形向「是」分支（标签「是」）连到方块（标 Act 阶段）：「(Act) 执行工具（若工具是子智能体 → 递归触发其 ReAct 循环）」。
5. 箭头向下到方块（标 Observe 阶段）：「(Observe) 工具结果写回 memory」。
6. 从该方块画一条明显的**回环箭头**向上回到第 2 步「(Reason) 调 LLM」节点，形成闭环（可在回环线上标「循环」）。
7. 从菱形向「否」分支（标签「否」）连到方块：「生成最终回复 Msg」。
8. 箭头向下到结束圆角矩形：「返回调用方」。

视觉强调：Reason / Act / Observe 三个阶段用不同柔和底色区分，循环回环箭头醒目（用强调色）。底部可加一行小字注释「当“工具”是子智能体时，(Act) 触发子智能体一整轮完整 ReAct 循环——递归嵌套」。

要求：竖版构图，箭头流向清晰，循环结构一眼可辨，专业技术文档插图质感，可加小标题「ReAct Agent 工作原理」。
